# Дорожная карта проекта IntelBot

Поэтапный план разработки и внедрения. Каждый этап завершается конкретным, проверяемым результатом.

---

## Этап 0. Подготовка и проектирование (1–2 недели) ✅
**Цель:** зафиксировать требования, утвердить архитектуру, подготовить окружение.

- [x] Интервью / формирование типового портрета клиента
- [x] Финализация технологического стека
- [x] Создание репозитория и структуры проекта (`CLAUDE.md`)
- [x] Настройка виртуального окружения, зависимости
- [x] `.env.example` и документация по запуску

**Результат:** утверждённое ТЗ, репозиторий с базовой структурой, готовое окружение.

---

## Этап 1. Ядро системы: парсинг и хранение данных (3–4 недели) 🔄
**Цель:** создать модули сбора информации и сохранять её в структурированном виде.

- [x] Модели SQLAlchemy (`Client`, `Source`, `News`, `Settings`, `Feedback`)
- [x] `database/db.py` — async подключение, сессии
- [x] `config.py` — AppSettings + `load_client_configs()`
- [ ] `database/crud.py` — CRUD-операции
- [ ] `parsers/base.py` — абстрактный базовый парсер
- [ ] `parsers/rss.py` — RSS-парсер (feedparser)
- [ ] `parsers/telegram.py` — Telethon-парсер (публичные каналы)
- [ ] `parsers/website.py` — aiohttp + BeautifulSoup4
- [ ] `parsers/social.py` — заглушка
- [ ] Генерация эмбеддингов (sentence-transformers, мультиязычная модель)
- [ ] Интеграция ChromaDB: коллекции per client
- [ ] `scheduler.py` — APScheduler, периодический запуск парсеров
- [ ] Сохранение сырых новостей в `data/clients/<id>/raw/`

**Результат:** парсеры работают, новости в SQLite + JSONL, эмбеддинги в ChromaDB, планировщик запущен.

---

## Этап 2. Умный анализ: RAG и генерация контента (3–4 недели)
**Цель:** подключить локальную LLM, реализовать RAG-пайплайн.

- [ ] Ollama + модель `saiga_llama3_8b` — установка и тестирование
- [ ] `processors/embeddings.py` — генерация эмбеддингов
- [ ] `processors/vector_store.py` — работа с ChromaDB
- [ ] `processors/rag.py` — поиск k ближайших соседей + формирование few-shot контекста
- [ ] `processors/llm.py` — вызов Ollama, промпт, парсинг ответа
- [ ] `processors/deduplicator.py` — hash-check + TF-IDF cosine similarity
- [ ] Сохранение `summary`, `sentiment`, `entities`, `hashtags` в таблицу `News`
- [ ] (Опционально) slovnet для NER как дополнительный источник

**Результат:** новости автоматически получают саммари, тональность и теги через RAG + LLM.

## Этап 3. Telegram-бот: интерфейс и доставка (2–3 недели)
**Цель:** разработать удобный интерфейс: регистрация, настройки, получение новостей, обратная связь.

- [ ] `bot/handlers/start.py` — приветствие, краткое описание
- [ ] `bot/handlers/sources.py` — просмотр и управление источниками
- [ ] `bot/handlers/settings.py` — ключевые слова, частота уведомлений
- [ ] `bot/handlers/last.py` — последние N новостей
- [ ] `bot/keyboards.py` — inline-клавиатуры
- [ ] `bot/middlewares.py` — проверка доступа, логирование
- [ ] Отправка новостей: форматирование сообщения (саммари, теги, кнопки)
- [ ] Кнопки под сообщением: 🔗 Перейти / 📌 Сохранить / ❌ Не интересно
- [ ] Сохранение feedback в БД

**Результат:** рабочий бот с командами, уведомлениями и сбором реакций.

---

## Этап 4. Мультиарендность, администрирование и onboarding (4–5 недель)
**Цель:** обслуживание нескольких клиентов, автоматизация подключения через MCP + Google Forms.

- [ ] 4.1. Все операции привязаны к `client_id`
- [ ] 4.2. Изоляция данных: `data/clients/<id>/` (конфиг, сырые новости, Chroma)
- [ ] 4.3. Админ-команды (доступ по `ADMIN_IDS`):
  - `/add_client` — ручное добавление
  - `/list_clients` — статус клиентов
  - `/update_client <id>` — обновление конфига
- [ ] 4.4. Автоматизация onboarding (MCP + Google Forms):
  - Настройка MCP-сервера для Google Sheets (сервисный аккаунт уже сохранён)
  - Шаблон опросного листа (Google Форма)
  - `/create_questionnaire <client_name>` → копия формы → ссылка клиенту
  - Мониторинг ответов → генерация JSON-конфига → запись в БД → активация
- [ ] 4.5. Динамическая перезагрузка конфигов (без перезапуска бота)

**Результат:** несколько клиентов одновременно, автоматический onboarding через форму.
